# Getting help with CleanSchema

Thanks for using CleanSchema! Here's how to get help:

## 📚 Start here

1. **Read the [README](./README.md)** — install, flow, detection taxonomy, and privacy guarantees
2. **Check [CHANGELOG.md](./CHANGELOG.md)** — what changed in the version you're running
3. **Skim open [issues](https://github.com/rjboogey/cleanschema/issues)** — your question may already be answered

## 💬 Ask questions

- **General how-to, ideas, show-and-tell**: [GitHub Discussions](https://github.com/rjboogey/cleanschema/discussions) — the best place for open-ended questions
- **Report a bug**: [open an issue](https://github.com/rjboogey/cleanschema/issues/new?template=bug_report.yml) — please use the template
- **Request a feature**: [open an issue](https://github.com/rjboogey/cleanschema/issues/new?template=feature_request.yml) — include the real-world use case

## 🔒 Security

Found a way for data to leak? **Please email security@drjonesy.com** rather than opening a public issue. See [SECURITY.md](./SECURITY.md) for details and disclosure policy.

## 🏢 For businesses

CleanSchema is MIT-licensed — use it freely at your company. If you need:

- Priority support
- A custom detection ruleset (industry-specific regulated data types)
- Integration help with your data pipeline
- An air-gapped deployment walkthrough
- Training for your team on data-minimization practices

…reach out to [drjonesy.com/consulting](https://drjonesy.com/consulting).
